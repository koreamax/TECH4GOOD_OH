## 디자인 토큰 (시안 추출)

- Primary(민트그린) `#5FC98E`, CTA·확인 버튼 배경 / 흰 텍스트
- 다크 필 토글: 검정 배경 캡슐, 선택=흰 배경 칩, 비선택=흰 텍스트
- 카드: 흰 배경, radius 16, 연회색 보더, 사진 상단 꽉 채움
- 강조(신고 수치): 오렌지레드 계열
- 지도: 경로 = 점선 폴리라인, 시작점 = 녹색 도트, 현재 위치 = 강아지 사진 원형 핀

## 핵심 사용자 플로우 (데모 시나리오)

1. 홈 → [오늘의 산책 시작하기!] → "카메라가 켜졌어요!" 모달 [확인했어요] → 산책 시작.
2. 지도 뷰에 점선 경로·강아지 핀, 시간·거리 누적. 토글로 카메라 뷰 전환 가능.
   AI가 프레임 분석(모델 미탑재 시 목 탐지 모드 — 동일 UX).
3. 파손 탐지 + 신뢰도 임계값(기본 0.60) 이상 → 진동 + 알림 카드 → 탭하면 상세.
4. 상세에서 캡처·유형·신뢰도·주소·AI 민원 문안 확인 →
   [신고하기] = 모의 접수(접수번호) → 신고 완료 화면 → [산책 계속하기].
   [아니에요] = 오탐 저장(재학습 데이터).
5. [산책 종료] → 요약 카드(경로·통계·사진 첨부) → 저장 → 홈/기록/신고 현황에 반영.

## 탐지 알람 정책 (선 승인 구조)

- 신뢰도 < 임계값: 알람 없음, 지도 데이터로만 조용히 축적(서버 전송).
- 신뢰도 ≥ 임계값: 진동 + 알림 카드. **사용자가 [신고하기]를 눌러야만 접수.**
- 동일 지점(반경 30m) 중복 탐지는 신뢰도 점수 가중 합산 (ARCHITECTURE.md).
